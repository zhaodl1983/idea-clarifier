# Amazon Working Backwards (亚马逊逆向工作法)

## 1. 核心价值理念 (Core Value)
亚马逊的所有创新都从**客户**开始，然后**向后推导**。通过在编写第一行代码前先编写新闻稿（Press Release），迫使团队明确产品的核心价值和客户痛点。

## 2. 内部新闻稿 (The Press Release / PR)
新闻稿应简洁明了，描述产品在推出之日的情况：
- **标题 (Heading)**：用读者能理解的方式命名产品（例如，“亚马逊推出单线程团队管理工具”）。
- **副标题 (Sub-heading)**：一句话说明产品针对的客户群以及该客户群将获得的收益。
- **摘要 (Summary)**：描述产品及其核心收益。
- **问题 (The Problem)**：从客户的角度描述该产品试图解决的问题。
- **解决方案 (The Solution)**：描述该产品如何解决上述问题。
- **高管致辞 (Internal Quote)**：由产品负责人发表一段话，说明开发该产品的原因及对客户的期望。
- **客户体验 (Customer Quote)**：描述一个假想客户在实际使用产品后的受益感受。
- **行动呼吁 (Call to Action)**：告诉客户如何开始使用产品。

## 3. 常见问题解答 (FAQ)
PR 之后必须紧跟 FAQ，分为两个层面：
- **外部 FAQ (External FAQ)**：客户可能会问的问题（如“这个产品多少钱？”、“我现有的数据如何迁移？”）。
- **内部 FAQ (Internal FAQ)**：利益相关者会问的问题（如“这个产品的商业模式是什么？”、“对我们现有的服务有哪些依赖？”）。

## 4. 价值定义审计 (Value Definition Audit)
- **是否足够大胆？** 产品是否真正解决了客户的一个重大问题？
- **是否足够清晰？** 即使是非专业人士也能通过 PR 读懂产品的核心价值吗？
- **是否具备竞争优势？** PR 中是否体现了该产品相较于现有方案的独特价值点？
